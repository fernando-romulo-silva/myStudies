When changes made to a Docker image are ready to be published, which of the following 
commands would make a new image available, called ‘httpd:v2’?

A)
docker checkin httpd:v2

B)
docker tag httpd:v2 httpd:latest

C)
docker tag httpd:v2 httpd:latest

D)
docker tag httpd:latest httpd:v2 




































Correct answer is D

To create a tag TARGET_IMAGE that refers to SOURCE_IMAGE use:

$ docker tag SOURCE_IMAGE[:TAG] TARGET_IMAGE[:TAG]

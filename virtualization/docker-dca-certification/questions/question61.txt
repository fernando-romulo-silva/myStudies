Which labels are required to deploy a service with Interlock?


a) com.docker.lb.port

b) com.docker.interlock.port

c) com.docker.interlock.hosts

d) com.docker.lb.backend























61 - Answer(s) 'a'

Interlock is an application routing proxy service for Docker. 
Fully integrates with Docker (Swarm, Services, Secrets, Configs) Enhanced configuration 
(context roots, TLS, zero downtime deploy, rollback) Support external load balancers (nginx, haproxy, etc) via extensions.

$ docker service create \
 --name demo \
 --network demo \
 --detach=false \
 --replicas=5 \
 --label com.docker.lb.hosts=demo.local \
 --label com.docker.lb.sticky_session_cookie=session \
 --label com.docker.lb.port=8080 \
 --env METADATA="demo-sticky" \
 ehazlett/docker-demo

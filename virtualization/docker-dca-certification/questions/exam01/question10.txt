Paul have some containerized software that needs to have a reference to the hostname of the node that the software is running on. 
Which of the following commands Paul will pass the node hostname as an environment variable into each task in a service?

​
A)
docker service create --env NODE_HOSTNAME="{{.Node.Hostname}}" nginx
​
B)
docker service create -e NODE_HOSTNAME nginx
​
C)
docker service create --env NODE_HOSTNAME="{{Hostname}}" nginx
​
D)
docker service create --pass-node-hostname=true nginx






































The correct option is A

Service Constraints

node.id	-> Node ID
node.hostname -> Node hostname
node.role -> Node role (manager/worker)
node.platform.os -> Node operating system	node.platform.os==windows
node.platform.arch -> Node architecture	node.platform.arch==x86_64
node.labels -> User-defined node labels node.labels.security==high
engine.labels -> Docker Engine’s labels

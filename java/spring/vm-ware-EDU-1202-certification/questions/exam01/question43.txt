Which of the following are valid ways of adding a Bean definition to the IoC Container?

A)
Calling DefaultListableBeanFactory.registerBeanDefinition

B)
<bean/> in XML
​
C)
@Bean annotated method in a @Configuration class
​
D)
Java Class annotated with @Component





































Answer 'A', 'B', 'C' and 'D'

All of these are valid ways of adding a bean definition to the IoC Container. 

https://docs.spring.io/spring/docs/current/spring-framework-reference/core.html#beans-definition

A)

public class GenericBeanDefinitionExample {

  public static void main (String[] args) {
      DefaultListableBeanFactory context =
                new DefaultListableBeanFactory();

      GenericBeanDefinition gbd = new GenericBeanDefinition();
      gbd.setBeanClass(MyBean.class);

      MutablePropertyValues mpv = new MutablePropertyValues();
      mpv.add("date", new Date());

      //alternatively we can use:
      // gbd.getPropertyValues().addPropertyValue("date", new Date());
      gbd.setPropertyValues(mpv);

      context.registerBeanDefinition("myBeanName", gbd);

      MyBean bean = context.getBean(MyBean.class);
      bean.doSomething();
  }
}

B)

<beans>

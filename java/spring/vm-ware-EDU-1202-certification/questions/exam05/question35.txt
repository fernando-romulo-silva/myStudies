Why is field-based dependency injection not recommended?

A)
It hides the dependencies

B)
Because it does not use Class-based Dependency Injection

C)
Because it uses the Java Reflection API, which may adversely impact performance

D)
It disallows final/immutable field declaration











































Answers 'A' and 'D' 

Concerns regarding Reflection API performance are unfounded, as it is used in Dependency Injection anyways.

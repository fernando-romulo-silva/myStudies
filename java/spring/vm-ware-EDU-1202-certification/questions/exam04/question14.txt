In Spring Security, how can SpEL-based authorization constraint annotations be enabled?

A) They are enabled by default.
​
B) @EnableGlobalMethodSecurity(jsr250Enabled = true)
​
C) @EnableGlobalMethodSecurity(prePostEnabled = true)

D) @EnableGlobalMethodSecurity(securedEnabled = true) 









































Answers 'C'

When set to true, the attribute prePostEnabled enables the annotations @PreAuthorize, @PostAuthorize, @PreFilter and @PostFilter, 
all of which may use SpEL expressions. 

@Configuration
@EnableGlobalMethodSecurity(
    securedEnabled = true, // enable global security
    prePostEnabled = true, // prePostEnabled enables the annotations @PreAuthorize, @PostAuthorize, @PreFilter and @PostFilter
    jsr250Enabled = true)  // Support for JSR-250 annotations can be enabled using, @PostContruct, @PreDestroy and  @Resource 
public class MethodSecurityConfig
    extends GlobalMethodSecurityConfiguration {
}
